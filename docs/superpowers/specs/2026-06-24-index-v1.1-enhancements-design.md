## Component 1 -- Cycle detection

**New module:** `src/index_graph/graph/cycles.py`.

Today `viz/layout.py` computes "back-edges" as a *layout* heuristic (edges pointing upward in the role layering). That is not true cycle analysis. This component adds real analysis over the directed edge set.

**Interface:**
```python
# graph/cycles.py
def find_cycles(edges: Sequence[Edge]) -> list[tuple[str, ...]]:
    """Return the node sets of each dependency cycle, deterministically.

    Uses Tarjan's SCC over internal repo->repo edges. Each strongly-connected
    component with >1 node (or a self-loop) is a cycle. Returns a sorted list
    of sorted node-name tuples. A pure DAG returns []."""

def cycle_edge_keys(edges: Sequence[Edge], cycles: Sequence[tuple[str, ...]]) -> set[tuple[str, str]]:
    """The (from, to) keys of edges whose endpoints are in the same cycle."""
```

- Only **internal** edges participate (external targets can't form repo cycles).
- Deterministic: SCCs sorted by their first (alphabetically least) node; nodes within a cycle sorted.

**Wiring:**
- `graph/build.py`: the built graph object exposes `cycles: list[tuple[str, ...]]` and the `cycle_edge_keys` set (computed once, reused).
- `context/pack.py`: the pack gains a top-level `cycles` field (list of node-name lists), and **each relation entry in the pack gains `in_cycle: bool`** (the single source the dashboard reads -- the frozen `Edge` dataclass is left unchanged). `index context` and the dashboard both consume the pack.
- `cli.py`: `index graph --cycles` prints a human report ("2 cycles found: [shared-utils, config-loader] â€¦" or "no cycles -- clean DAG"); `index graph --json` includes `cycles`. Exit 0 always (reporting, not gating -- gating is `diff`'s job).

**Tests** (`tests/test_cycles.py`): empty graph â†’ []; pure DAG â†’ []; one 2-node cycle; one 3-node cycle; a self-loop; two disjoint cycles; determinism (shuffled input â†’ identical output); `cycle_edge_keys` correctness. Plus a `cli` test that `--cycles` reports them.

## Component 3 -- `index diff`

**New module:** `src/index_graph/diff.py`. **New subcommand:** `index diff`.

**Interface:**
```python
# diff.py
@dataclass(frozen=True)
class GraphDiff:
    repos_added: tuple[str, ...]
    repos_removed: tuple[str, ...]
    edges_added: tuple[tuple[str, str], ...]      # (from, to)
    edges_removed: tuple[tuple[str, str], ...]
    edges_changed: tuple[tuple[str, str, str, str], ...]  # (from, to, old_conf, new_conf)
    def is_empty(self) -> bool: ...               # True iff no drift

def diff_graphs(old: dict, new: dict) -> GraphDiff:
    """Diff two `index graph --json` outputs (nodes + edges). Deterministic;
    all tuples sorted."""
```

- **CLI:** `index diff <baseline.json> <current.json> [--json] [--exit-zero]`.
  - Reads two graph-JSON files. Prints a human report (added/removed repos, added/removed edges, confidence changes) or `--json`.
  - **Exit code:** `1` if any drift (default -- so CI fails on unexpected structural change), `0` if identical. `--exit-zero` forces `0` (report-only).
- Edge identity = `(from, to)`. "Changed" = same `(from, to)`, different confidence.
- Documented use: commit a baseline `index graph --json > deps.json`; in CI run `index graph --json > new.json && index diff deps.json new.json` â†’ fails on drift. A ready-to-paste GitHub-Action snippet goes in the README.

**Tests** (`tests/test_diff.py`): identical â†’ `is_empty()` + exit 0; added edge; removed edge; confidence change; added repo; removed repo; combined; `--json` shape; `--exit-zero` overrides exit. A `cli` test for the exit codes.

## Error handling / risks

- **Tooltip/JS injection:** all tooltip content routes through the existing HTML-escape path; covered by the existing escaping tests + a new one.
- **Cycle detection on large graphs:** Tarjan's is linear; 101-repo graphs are trivial. No perf concern.
- **`diff` on mismatched schemas:** if a JSON lacks expected keys, fail with a clear `SystemExit` message (not a traceback).
- **Demo git-repo fixture:** nested `.git` dirs can't be committed; the fixture is generated in a temp dir at build time, only the rendered HTML + PNG are committed. The build script is deterministic given a fixed sample definition (timestamps stamped post-render or normalized).
- **Pages exposure:** only the synthetic demo + the public package ship; the build script contains no real paths.

## Success criteria

1. `index graph --cycles` reports true cycles (verified against a known-cyclic fixture); a DAG reports none. Determinism holds.
2. The rendered dashboard shows edge-evidence on hover, a legend, neighborhood highlight on node focus, and visibly distinct cycle edges -- asserted in tests + visible in the screenshot.
3. `index diff baseline.json current.json` reports drift and exits `1`; identical inputs exit `0`; `--exit-zero` overrides.
4. A live, clickable dashboard is reachable at the GitHub Pages URL (synthetic sample only); the README leads with a real screenshot + the live link; the repo description/topics/homepage are updated.
5. All prior tests stay green; new tests cover cycles, diff, and the viz additions; suite â‰Ą ~150.
